# 更新日志

## 0.2.1 - 2026-05-27

- 修复右侧 `CLIP` 数字框被清空后生成时报 `Failed to convert an input value to a FLOAT value` 的问题；现在空值会自动回到 `1`。
- 加强工作流保存兜底：保存时会把异常的 LoRA 默认 CLIP 权重修正为数字，旧工作流或误操作不容易再保存出空 FLOAT。
- 后端也增加兜底：即使外部工作流传入空值或非数字值，也会按 `1.0` 处理。
- 修复设置里的字体大小不明显生效的问题：按钮、输入框、提示词标签、LoRA 卡片等主要文字现在会一起跟随字体大小变化，默认字体大小为 `12px`。
- 批量添加选中的 LoRA 到正向提示词后，会自动清空选择状态，绿色选中框和勾选框不会继续残留。
- 补充用户说明：明确 v0.2.0 已修复 LoRA 接入误报、刷新后提示词错位/不能保存、自定义标签增删改、多选拖拽等反馈，并写清楚多个提示词标签如何一起拖动。

## 0.2.0 - 2026-05-27

- 修复 WebUI 接入后的 LoRA 误报缺失问题：现在会同步 WebUI/配置里的 LoRA 路径，并支持 `.lnk` 快捷方式、文件名、路径名和部分 metadata 别名匹配。
- 修复刷新页面后 Prompt / Negative prompt / CLIP 等字段错位的问题，减少“正向词跑到反向词里”和提示词无法保存的情况。
- 优化 LoRA 卡片库：入口改为 `快速添加 LoRA`，支持勾选多个 LoRA 后一次添加到正向提示词，并显示添加成功提示。
- 优化 LoRA 面板布局：折叠后只保留一条紧凑栏，拖动分隔线时不再把 LoRA 区压成一坨或撑出大片空白。
- 优化提示词输入：底部输入框更明显，按 Enter 添加时默认走翻译；输入过程中不再刚打一个字就自动翻译。
- 新增自定义提示词标签管理，可以新增、修改、删除自己的标签内容。
- 新增多选拖拽提示词标签：按住 Ctrl 多选后可以一起移动。
- 修正 LoRA 权重编辑：现在会正确修改 `<lora:name:weight>` 里的权重，而不是把整段 LoRA 标签当普通提示词加权。
- 已实测从浏览器工作流切换并生成成功：`waiANINSFWPONYXL_v60.safetensors` 整合 XL 模型、分体 `Anima/Qwen` 模型，以及从 Anima 切回 XL 后再次生成。

## 0.1.9 - 2026-05-27

- 优化 WebUI 配套扩展入口：已经安装 Prompt All in One / TagComplete 的用户会看到“扩展已存在”，不再提示重复下载。
- 修正前后端版本不一致时的报错说明：如果补齐扩展接口返回 404 / 405，会提示重启 ComfyUI 加载新的后端接口。
- 一键接入 WebUI 会同步返回配套扩展检测结果，方便用户判断是“直接接入”还是“补齐缺失项”。

## 0.1.8 - 2026-05-27

- 新增无 WebUI 独立模式：没有本机 WebUI 时也会提供内置基础 tag 分类、自动补全、常用反向词和简单中英映射。
- 新增首次启动向导：可选择连接 WebUI、使用内置数据或导入本地词库。
- 新增集中设置面板：管理数据来源、翻译来源、布局尺寸、Tag 显示方式和 LoRA 卡片大小。
- 新增词库导入接口，支持从 JSON / CSV / TSV 导入自定义 tag 映射。
- 新增配套数据补齐：已有 WebUI 和扩展时无需下载；WebUI 缺扩展时可只补齐缺失项；没有 WebUI 时可下载到本节点 data 目录直接使用。

## 0.1.7 - 2026-05-26

- 优化提示词面板布局调整：Prompt 文本框、已输入 tag 区、Prompt All in One 标签区和 Extra Networks 的高度都可以更方便地拖动分配。
- 修复 Negative prompt 折叠后仍然占用大量纵向空间的问题。
- 修复提示词 tag 编辑工具条在底部或右侧滚动条附近被遮挡的问题。
- 补充说明已有操作：双击提示词 tag 可以隐藏/禁用该 tag，再次双击可以恢复；被禁用的 tag 不会参与生成。
- 下个版本预告：会增加更详细的设置功能，集中管理翻译、布局、标签显示、LoRA 卡片和本机 WebUI 接入选项。
