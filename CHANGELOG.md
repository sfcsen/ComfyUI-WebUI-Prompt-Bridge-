# 更新日志

## 0.4.5 - 2026-06-20

- 修复部分工作流重新打开后，WebUI Prompt Bridge 节点界面可能只剩侧栏、提示词区域消失或出现大片空白的问题。
- `恢复尺寸` 和 `默认尺寸` 现在会同时清理异常布局缓存，遇到界面错位时可以直接恢复，不需要删除节点重连。
- 打开旧工程时会自动修复异常保存的 Bridge 节点尺寸，降低下次打开再次错位的概率。
- 修复导入提示词时可能出现 `Cross-origin requests are not allowed` 的问题，`localhost` 和 `127.0.0.1` 同端口访问会按同源处理。
- 优化导入、设置、AI、词库和资源下载等接口的错误提示，不再把原始 JSON 直接显示给用户。
- 补强图片输入节点和本地布局缓存的稳定性，减少浏览器缓存异常导致的界面中断。

## 0.4.4 - 2026-06-15

- 修复在 ComfyUI `设置 - 布局/显示` 中切换布局尺寸后，WebUI Prompt Bridge 节点可能持续闪烁或反复重排的问题。
- 节点尺寸、区域高度和布局缓存现在会跳过无变化写入，避免保存布局后重复触发 `setSize` 和自适应布局循环。
- 布局自适应调度增加合并保护，同一帧只执行一次重算，切换默认、紧凑、宽松、正向聚焦和极简 LoRA 布局时更稳定。
- 优化节点操作手感：顶部按钮、尺寸快捷和布局预设按钮更容易点击，键盘焦点更清楚，触控/鼠标操作更稳。
- 拖拽分隔条和右下角整体缩放手柄新增拖动中的视觉反馈，并补充可访问性标签。
- 布局预设按钮改为自动换行，窄侧栏或低缩放画布下不再硬挤成一排。

## 0.4.3 - 2026-06-15

- 修复提示词 tag 操作会吞掉换行/空行的问题；删除、拖拽、改权重后会尽量保留原文本里的换行分隔，避免多人提示词被压成一行。
- tag 分块区域新增可见的 `↵ / 换行符` 分块，换行符会像普通 tag 一样显示，方便确认提示词结构。
- 点击 `换行符` 分块可以删除对应换行，工具条里的 `↵` 按钮会真正把当前位置改成换行分隔。
- 换行符分块使用独立样式和文案，不再复用普通 tag 的中文翻译行，避免显示成“发饰”“看着你”等相邻 tag 的本地名。
- 已通过 ComfyUI API 使用 `WebUIPromptBridge` 节点提交多组带真实换行的多人提示词，历史记录中 `positive_prompt` 保留 `\n`，并成功生成输出图。
- 新增主节点内置图生图和局部重绘面板：可以直接在 WebUI Prompt Bridge 侧栏上传图片、开启图生图输入、调整重绘幅度，并在涂抹编辑器里绘制局部重绘 mask。
- 新增“一键接入图生图链路”：自动创建 VAE Encode 或 VAE Encode For Inpaint，并把 Bridge 的 image / mask 输出接到 KSampler 的 latent_image，减少手动接线出错。
- 主节点新增 image、mask、img2img_denoise 和 img2img_mode 输出，可用于自定义图生图、局部重绘和外部修复流程。
- 新增首次添加节点教程：第一次新建 WebUI Prompt Bridge 主节点会显示完整图文说明；用户也可以在设置里改为每次显示、关闭自动显示，或随时手动查看。
- 扩充提示词市场：新增 Danbooru + e621 合并词库、e621 SFW 拟人词库、e621 NSFW 零过滤/微过滤词库、Derpibooru 角色词库、Krea / Open Prompts 完整案例，以及 PromptHero、Civitai、Lexica、OpenArt 等案例入口。
- 提示词市场会显示过滤档位和导入数量，NSFW 来源提供“零过滤”和“微过滤”两种选择，便于按创作需求导入。
- 设置页的资源下载状态更醒目：下载、检查、成功和失败都会有明确高亮提示，避免误以为按钮没有响应。
- 修复节点内上传图片后预览地址错误导致“载入失败”的问题；上传图片现在会保存到 ComfyUI input 目录并正常预览。
- 图生图输入新增显式启用开关；未启用时不会把上传图片误接入生成流程。

## 0.4.0 - 2026-06-12

- 设置窗口改为更宽、更清晰的分组设置中心，新增主节点内置功能和高级扩展模块的显示开关，隐藏功能只影响界面，不会删除工作流里的参数。
- 主节点内置更多 WebUI 风格能力：区域提示词文本、区域 conditioning、负向 Common、区域翻转、区域 Preset、模型切换、尺寸快捷、布局预设和显示隐藏都可以直接在 Bridge 节点内配置。
- 新增高级扩展模块入口：ADetailer、ControlNet、SAM/Inpaint、Mask 区域和放大修复可以按需显示，并提供“一键构建外置节点”按钮，自动创建对应节点并连接 Bridge 已知输出。

- 新增后端高级模块节点：ControlNet Apply、Image Upscale、Latent Upscale、Set Latent Mask、ADetailer Conditioning/Apply、Inpaint Conditioning，并增加模块资产检测和按需下载入口。
- 修复隐藏/显示侧边栏会强制挤压提示词区域的问题；新建节点默认显示侧边栏，旧的折叠状态不再污染新节点。
- 优化宽松布局，让长提示词、LoRA 浏览器和侧栏功能有更充足的可视空间；应用布局预设时会清理旧的高度缓存，减少“之前保存过的布局覆盖新预设”的情况。
- 同步更新推荐工作流和 3 个最小实例工作流里的 Bridge 节点 schema，补齐新增输入和 `module_config` 输出，旧示例打开后不再依赖前端自动补字段。
- 为 Bridge 主节点和高级模块节点增加搜索别名，用户可以在节点库里搜索 `webui`、`prompt bridge`、`区域提示词`、`ADetailer`、`ControlNet`、`Hires fix` 等关键词找到节点。
- 文档补充本次新增功能、默认显示策略、高级扩展用途和按需下载说明；ADetailer 明确说明为脸部、手部和人物局部细节修复工具。
