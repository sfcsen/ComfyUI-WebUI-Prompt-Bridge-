# ComfyUI WebUI Prompt Bridge v0.2.1

这是 v0.2.0 的补丁更新，主要修复右侧 `CLIP` 数字框为空时生成失败的问题，并补充用户操作说明。

## 本次修复

- **修复 CLIP 空值报错**：如果右侧 `CLIP` 数字框被清空，生成时可能出现 `Failed to convert an input value to a FLOAT value`。现在空值会自动回到 `1`。
- **保存更稳**：保存工作流时会把异常的 LoRA 默认 CLIP 权重修正成数字，避免旧工作流或误操作保存出空 FLOAT。
- **后端兜底**：即使外部工作流传入空值或非数字值，也会按 `1.0` 处理。
- **字体大小真的生效**：设置里的字体大小现在会影响主要按钮、输入框、提示词标签、LoRA 卡片等文字，默认字体大小为 `12px`。

## 已确认修复的用户反馈

- **WebUI 接入后 LoRA 误报缺失**：会同步 WebUI/配置里的 LoRA 路径，支持 `.lnk` 快捷方式、文件名、路径名和部分 metadata 别名匹配，减少“能看到预览和触发词，但点击 LoRA 提示未找到”的情况。
- **刷新后提示词错位和不能保存**：修复 Prompt / Negative prompt / CLIP 字段错位，减少正向词跑到反向词、提示词无法保存的问题。
- **自定义标签管理**：可以新增、修改、删除自己的提示词标签内容。
- **多个提示词一起拖拽**：按住 `Ctrl` 点选多个提示词标签，再拖动其中任意一个被选中的标签，松手后会一起移动到新位置。

## 升级提示

更新后请重启 ComfyUI，再刷新浏览器页面。

如果你使用 WebUI 目录接入 LoRA，建议重新点一次 `一键接入 WebUI`，让节点刷新 LoRA、checkpoint、VAE、embeddings 等路径。
