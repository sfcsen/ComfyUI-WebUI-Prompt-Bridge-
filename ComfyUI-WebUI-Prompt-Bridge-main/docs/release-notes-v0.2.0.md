# ComfyUI WebUI Prompt Bridge v0.2.0

这是一次面向实际使用体验的更新，重点修复用户反馈的 LoRA 接入、提示词保存、标签编辑和面板拖动问题。

## 主要更新

- **LoRA 接入更稳了**：接入 WebUI 后会同步 LoRA 路径，支持 `.lnk` 快捷方式、文件名、路径名和部分 metadata 别名匹配，减少“能看到预览和触发词，但提示 LoRA 未找到”的情况。
- **提示词保存更稳定**：修复刷新页面后 Prompt / Negative prompt / CLIP 字段错位的问题。
- **快速添加 LoRA**：LoRA 面板入口改为 `快速添加 LoRA`，可以勾选多个 LoRA 后一次添加到正向提示词，并显示添加成功提示。
- **LoRA 面板不再乱跳**：折叠后只保留一条紧凑栏，拖动分隔线时不会把 LoRA 区压成一坨，也不会留出大片空白。
- **输入体验更顺**：底部输入框更明显，按 Enter 添加时默认翻译；输入过程中不会刚打一个字就自动翻译。
- **自定义标签管理**：可以新增、修改、删除自己的提示词标签。
- **多选拖拽标签**：按住 Ctrl 多选提示词标签后，可以一起拖动排序。
- **LoRA 权重编辑更准确**：现在会修改 `<lora:name:weight>` 里的权重，不会把 LoRA 标签当普通提示词处理。

## 已验证

已经在浏览器里的真实 ComfyUI 工作流中验证：

- XL 整合模型 `waiANINSFWPONYXL_v60.safetensors` 可以正常生成。
- 分体 `Anima/Qwen` 模型可以正常生成。
- 从 Anima 切回 XL 后再次生成也正常。

## 升级提示

更新后请重启 ComfyUI，再刷新浏览器页面。如果页面还显示旧按钮或旧布局，通常是浏览器或 ComfyUI 前端缓存没有刷新。

如果你使用 WebUI 目录接入 LoRA，建议重新点一次 `一键接入 WebUI`，让节点刷新 LoRA、checkpoint、VAE、embeddings 等路径。
