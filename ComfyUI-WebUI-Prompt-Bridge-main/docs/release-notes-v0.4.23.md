# ComfyUI WebUI Prompt Bridge v0.4.23

本次更新解决完整词库接入后仍显示精简数据、市场词条只能逐条删除，以及新节点载入时明显跳动和底部留白的问题。

## 更新内容

- 接入的 WebUI 缺少 Prompt All in One 或 TagComplete 时，自动使用插件本地已下载的完整数据包。
- 提示词分类新增 `全库搜索`，按需搜索完整 TagComplete 数据，避免一次渲染十几万条标签。
- `设置 -> 编辑词库` 新增 `清空自定义/市场`，确认后一次清空全部导入词条。
- 清空操作不会删除 WebUI 原始词库、Prompt All in One 分类数据或用户收藏。
- 设置页显示分类词库和全库搜索的实际来源。
- 新节点首帧直接匹配最近使用的布局预设；默认紧凑布局从创建到加载完成始终保持 `920×680`。
- 保存设置、画布缩放或刚放下节点后会自动收口 LoRA 区域，避免节点底部出现大块空白。
- 最小节点宽度从 `840px` 调整为 `640px`，窄版自动换行并把 LoRA 控件改为单列排列。
- 远景摘要默认在画布缩放低于 `50%` 时出现；可在 `设置 -> 布局 / 显示` 中调整 `0-100%` 阈值，设为 `0` 可关闭。

## 验证

- 验证 WebUI 扩展缺失但本地数据包存在时，分类和补全均选择本地完整数据。
- 验证完整 TagComplete 搜索可以返回标签并加入提示词。
- 验证批量清空只移除自定义/市场词条，并同步清理市场导入记录。
- 验证新节点首帧尺寸不跳变、标题栏拖动距离完整，拖动结束后位置与尺寸不再漂移。
- 验证新建、保存设置和缩放往返时 LoRA 底部空白均在稳定阈值内，节点外框始终保持 `920×680`。
- 验证 `640px` 节点内的顶部工具、提示词和 LoRA 区域没有横向溢出。
- 验证默认 `50%` 边界、自定义阈值保存，以及保存后立即切换摘要模式。
- 通过 Python、JavaScript 语法检查和前端兼容回归。

更新后请重启 ComfyUI，并强制刷新浏览器页面。

本版本会同步发布到 ComfyUI Registry，包版本为 `0.4.23`。
