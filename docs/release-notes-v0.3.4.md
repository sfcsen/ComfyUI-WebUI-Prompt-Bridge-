# ComfyUI WebUI Prompt Bridge v0.3.4

这是 `v0.3.3` 之后的界面补丁，重点修复节点隐藏侧栏后的宽度自适应问题，并增加误缩后的恢复入口。

## 更新内容

- 修复点击节点顶部 `隐藏侧栏` 后，Bridge 节点内部右侧仍留下大片空白的问题。现在隐藏侧栏会自动收窄节点，提示词、标签和 LoRA 区域会贴合可用宽度。
- 再次点击 `显示侧栏` 会恢复隐藏前的节点宽度，适合在“连线模式”和“完整控制模式”之间来回切换。
- 提高节点最小安全宽度，避免旧工作流或误操作把节点压成一条窄栏，出现“只能拉长、拉不宽”的情况。
- 顶部主控新增 `恢复尺寸`，节点显示异常时可以一键回到默认宽高。
- 顶部主控新增 `拖宽节点`，右下角缩放手柄不在屏幕里时，也可以直接从顶部把节点横向拉宽。
- 内置教程同步补充顶部恢复尺寸和拖宽节点的说明。

## 使用提示

- 想露出节点接口连线时，点击 `隐藏侧栏`，节点会自动收成更紧凑的提示词布局。
- 想回到带生成按钮、模型切换和区域控制的完整布局时，点击 `显示侧栏`。
- 如果节点被旧尺寸数据或误操作压得太窄，先点顶部 `恢复尺寸`；如果只想加宽，可以拖动顶部 `拖宽节点`。

## Comfy Registry

本版本将 `pyproject.toml` 版本更新为 `0.3.4`，并保留现有 `[tool.comfy]` 注册元数据。推送到 `main` 后，仓库中的 `Publish to Comfy Registry` GitHub Action 会触发 Comfy Registry 发布。

## 验证

- 前端 ES module 语法检查通过：`node --check web/webui_prompt_bridge.js`
- 浏览器实测通过：隐藏侧栏后节点从 `1180px` 自动收窄到 `760px`，提示词区域贴满可用宽度；再次显示侧栏后恢复到 `1180px`。
- 浏览器实测通过：强制压窄节点会回到安全宽度，顶部 `拖宽节点` 可以正常横向拉宽。

## 升级提示

更新后请重启 ComfyUI，并强制刷新浏览器页面，确保前端扩展脚本生效。
